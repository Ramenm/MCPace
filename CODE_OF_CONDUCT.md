# Code of Conduct

MCPace is intended to be useful infrastructure for people who build with MCP clients and servers. The project works best when discussion stays practical, respectful, and evidence-led.

## Expected behavior

- Be respectful to maintainers, contributors, users, and upstream MCP authors.
- Assume good intent, but keep claims grounded in code, traces, docs, or reproducible commands.
- Share enough detail for others to reproduce bugs without exposing secrets or private user data.
- Separate product ambition from proven behavior. Do not market preview lanes as supported runtime guarantees.
- Help keep the repository safe: avoid posting credentials, tokens, private file paths, or sensitive logs in public issues.

## Unacceptable behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publicly sharing another person's private information or credentials.
- Repeatedly derailing issues or pull requests away from the concrete technical scope.
- Pressuring contributors to bypass safety, release, or verification gates.

## Reporting

Report conduct problems to the repository maintainer. Do not post private personal details, private messages, credentials, or unresolved vulnerability details in public issues. Security-sensitive reports should use `SECURITY.md` instead of public issues.

## Enforcement

Maintainers may edit or remove content, close issues, reject pull requests, or block accounts when behavior makes collaboration unsafe or unproductive.
