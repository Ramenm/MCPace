# Tombstone Report

## Итог

В этом экспортном архиве **tombstone-файлы не использовались**.

Причина: удалённые элементы были либо runtime-артефактами, либо временными файлами, не требующими сохранения имени файла внутри полезного пакета.

## Что исключено из архива вместо tombstone

- `logs/` — runtime-логи, не source of truth
- `backups/` — архивы данных, не нужны в distributable package
- `data/` — runtime state и вложенный `.git`, не должны ехать в чистом архиве
- `.serena/` — tool-local metadata, не нужна для запуска пакета
- `tmp-runstack.ps1`, `tmp-status.ps1`, `tmp_poll.ps1`, `tmpdiag.ps1` — временные диагностические скрипты
- `__tmp_ensure6.ps1`, `__tmp_ensure7.ps1` — временные одноразовые скрипты
- `run-check.cmd`, `run-start.cmd` — дублирующие Windows wrappers без дополнительной логики

## Почему tombstone не понадобился

- удалённые файлы не импортировались из оставленного кода;
- build/test scripts пакета не зависят от них;
- документация переписана под чистый export без ссылок на исключённые артефакты.
