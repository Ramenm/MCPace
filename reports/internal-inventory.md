## Refactor rules

1. Do not add new default upstream servers in setup; import or install only when the user supplied a source.
2. Keep client config mutation in `src/client/actions/*`; setup may orchestrate it but should not rewrite client config directly.
3. Keep source-type normalization in `src/source_type.rs`; do not re-add local normalizers.
4. Keep MCP HTTP boundary checks centralized in `src/dashboard/http_boundary.rs` and response headers in `src/dashboard/response.rs`.
5. Split large runtime files only along existing seams: args/model/render/runtime/tests. Avoid parallel duplicate implementations.
