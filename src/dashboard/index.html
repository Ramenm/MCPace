Context

Help & setup guide

What to look at first, what is automatic, what is manual, and what should stay hidden until needed.

1. Reading order

Use the page in this order: Basics, runtime status, server rows. Setup tools and diagnostics are optional drawers.

First the five base checks. Second server rows: evidence, state, actions. Third setup tools only when importing, discovering, or wiring clients. Then details only when changing routing.

2. Dynamic fields

These change after refresh, Test, hub activity, or cache warmup.

Backend link, runtime readiness, active locks. tools/list evidence, test failures, cached tool counts. Recommended policy and server groups.

3. Stable fields

These are configuration and should not move around unexpectedly.

Server name, command, URL, source path. Env/header names are visible; values stay hidden. Enabled state and manual routing policy.

4. Setup options

Choose how much MCPace helps. The safe default is preview or save disabled, then test.

Import reuse an existing client config first. Manual paste a command or URL. Discovery preview known candidates first. Policy plan apply conservative routing after evidence. Overrides stay inside Details so routine setup stays small.
